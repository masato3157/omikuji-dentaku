# デザイン改良（画面仕様書対応）

画面仕様書に基づき、標準的な電卓デザインと独自の「いい加減な」演出を実装します。

## User Review Required

- **デザインの大幅変更**: 既存のポップでカラフルなデザインから、黒基調の標準的な電卓デザイン（iOS/Android風）に完全に刷新します。
- **アニメーション**: 計算実行時（`=`押下）に、計算結果エリアが上にスライドし、下から隠されたエリアが出現するアニメーションを追加します。

## Proposed Changes

### src/App.jsx

- **1. 初期画面（通常モード）**
  - **配色**: 背景 `bg-black`、文字 `text-white`。
  - **ボタン**: 円形 (`rounded-full`)。
    - 数字キー: ダークグレー (`bg-zinc-800`, `text-white`)
    - 機能キー(AC, ±, %): ライトグレー (`bg-zinc-300`, `text-black`)
    - 演算キー(÷, ×, -, +): オレンジ (`bg-amber-500`, `text-white`)
    - 0キー: 横長カプセル形状。
  - **レイアウト**: 画面下部にキーパッド、その上に結果表示。標準的なスマホ電卓の配置。

- **2. 計算実行時のアニメーション**
  - State: `isCalculated`フラグを追加。
  - Action: `=` を押すと `isCalculated = true`。
  - Effect: 結果表示エリア（とキーパッド全体？）が上にスライド (`transform: translateY(...)`)。
  - **音**: 「ガシャン」という機械音（今回は音ファイルがないため、視覚的な動きのみ先行実装、またはWeb Audio APIで簡易的な音を生成検討）。

- **3. 下段チラ見せエリア**
  - スライドによって空いた下部スペース（またはスライドアップしたコンテンツの下層）に配置。
  - **構成**:
    - 左側: カテゴリ名（小・灰）、チラ見せ一言（太字）。
    - 右側: オレンジ色の正方形ボタン「⤵」。

- **4. 詳細ポップアップ（モーダル）**
  - `showFullOmikuji` 時に全画面オーバーレイではなく、画面中央または下部からのモーダルとして表示。
  - **構成**:
    - カテゴリ名
    - **広告エリア**: `300px x 250px` の固定サイズコンテナ（背景グレーなど）。ここに `fakeAds` の内容を表示。
    - 続きの一言: 広告の下に表示。
    - 閉じるボタン: 最下部。
